# AGENTS.md

## このプロジェクトでの基本方針
- 勝手にコード編集を始めない
- デフォルトは設計相談、計画整理、レビュー、説明
- 実装は、こちらが明示的に依頼したときだけ行う

## 実装前に必ず整理すること
- 目的
- 責務の分割
- 状態管理
- success / failure / pending の分岐
- どの spec で確認するか

## レビュー時の優先事項
- 責務の置き場所が正しいか
- 状態管理に矛盾がないか
- 既存フローを壊していないか
- spec の観点が適切か

## 学習プロジェクトとしての扱い
- このリポジトリは学習用
- コーディング速度より理解を優先
- 概念の質問では、まず説明を行い、その後に必要なら実装案を出す

## 実装の許可条件
- 次のように明示されたときだけ実装してよい
  - 実装して
  - 差分を出して
  - コードを書いて
- 曖昧な依頼では、壁打ち・設計相談モードのままにする

## このプロジェクトで主に任せること
- 変更差分の整合確認
- ファイル前提の責務レビュー
- 破綻点の洗い出し
- spec 観点の確認

## なるべく避けること
- 基礎概念の長い説明のためにトークンを使いすぎること
- 明示依頼なしの自動実装
- 学習意図を無視して完成優先で進めること
